Architecture

System design, tech stack, infrastructure, data flow, and integrations across all 5 business verticals.

System Overview 5 Verticals · 1 Platform
The platform is the connective tissue of the entire padel ecosystem. Every vertical — club operations, franchise network, wellness, equipment, and tournaments — flows through a single unified system. The app is the player-facing layer. The dashboard is the operator layer. AWS UAE region ensures data residency compliance and low latency for GCC users.

Tech Stack Chosen for Speed, Scale & UAE Compliance
Mobile App
  • React Native — iOS + Android from one codebase
  • Expo for fast iteration and OTA updates
  • React Navigation for routing
  • Zustand for state management
  • Firebase Push Notifications
Backend API
  • Node.js + Express — REST API
  • JWT + OAuth2 for authentication
  • Prisma ORM for database access
  • Bull + Redis for job queues
  • WebSockets for live scores/brackets
Web Dashboard (Operator)
  • Next.js — Operator + franchise portal
  • Tailwind CSS for UI
  • Recharts for analytics
  • Role-based access (owner / manager / staff)
  • Real-time court availability view
Database & Storage
  • PostgreSQL — primary (RDS on AWS)
  • Redis — session cache + booking locks
  • Amazon S3 — media, profiles, content
  • ElasticSearch — player + club search
  • TimescaleDB — time-series analytics
Cloud Infrastructure
  • AWS Middle East (UAE) — data residency
  • ECS Fargate — containerised services
  • CloudFront CDN — global asset delivery
  • Route 53 — DNS + health checks
  • Auto-scaling on peak booking times
AI & Intelligence
  • Claude API — coaching insights + chat
  • Custom ML model — player matchmaking
  • Recommendation engine — session + wellness
  • Demand forecasting — court scheduling
  • Analytics pipeline via Segment + Mixpanel

Core Data Flows How the system works end-to-end
Court Booking Flow
Player opens app → Views real-time court availability → Selects slot (Redis lock prevents double-booking) → Payment via Telr → Booking confirmed (DB write) → QR code sent to player → Smart lock activated at court time
Matchmaking Flow
Player posts open game request → ML model scores skill-compatible players → Push notification sent to top matches → Player accepts → booking auto-created → Post-match result logged → ELO updated
Franchise Revenue Flow
Franchise location processes booking → Revenue logged in platform → Monthly royalty (6%) auto-calculated → Franchise dashboard updated → Royalty invoice generated → Stripe billing
Security & Compliance UAE Data Residency Required
Authentication & Access
  • JWT tokens with 15-min expiry + refresh tokens
  • OAuth2 social login (Google, Apple)
  • Role-based access: Player / Staff / Manager / Owner / Franchise
  • MFA for all operator accounts
  • API key rotation for franchise integrations
Data Protection
  • All data stored in AWS UAE region (PDPL compliance)
  • AES-256 encryption at rest
  • TLS 1.3 for all data in transit
  • Payment data handled by Telr (PCI-DSS, never stored)
  • GDPR-ready data export and deletion API
UAE Compliance
  • UAE PDPL (Personal Data Protection Law) compliant
  • Data localisation: all UAE user data stays in UAE region
  • Arabic language support in app and comms
  • UAE commercial licensing documentation ready
Infrastructure Security
  • WAF (Web Application Firewall) on all public endpoints
  • DDoS protection via AWS Shield
  • VPC with private subnets for all databases
  • Automated vulnerability scanning in CI/CD
Scalability — From 1 Club to GCC Network Multi-region ready
Year 1 — Single Club
  • Monorepo architecture, single deployment
  • ~2,500 MAU, low infra cost (~$800/mo AWS)
  • Single PostgreSQL instance (RDS)
  • Manual ops with lightweight tooling
Year 2 — Multi-Location
  • Microservices split: booking / wellness / commerce
  • Read replicas for DB as load increases
  • Franchise portal as separate service
  • ~20K MAU, scaled infrastructure
Year 3 — GCC Expansion
  • Multi-region deployment (UAE + KSA)
  • App B2B — white-label for external clubs
  • Full microservices, Kubernetes orchestration
  • ~75K MAU, enterprise-grade reliability
